
# 🦅 eBay 鹰眼 (Sleep Guard) 商业主控台架构文档 V1.5


## 二、 核心功能需求 (主打：高效管理 + 极致装逼)

### 1. 📊 数据大盘 (Dashboard - 核心装逼页)
* **实时监控墙**：高亮显示当前正在运行的监控任务数、今日抓取次数、今日拦截报警数。
* **大盘趋势图**：ECharts 渲染的折线图（展示最近 7 天全盘异常波动趋势，视频里看起来非常专业）。
* **滚动日志流 (Live Logs)**：页面右侧提供一个类似终端（Terminal）的黑色日志滚屏窗口，实时跳动 `[2024-05-xx 02:30:11] [INFO] Item 12345678 价格抓取正常: $19.99`。**这是抖音短视频最具视觉冲击力的部分。**

### 2. 👥 客户与通知管理 (CRM模块)
* **客户列表**：增删改查客户信息（姓名、微信号、订阅到期时间）。
* **报警路由配置**：为每个客户绑定其专属的推送通道（企微 Webhook URL、Bark ID、阿里云接收手机号）。支持一键测试发送。

### 3. 🎯 监控任务管理 (Task 模块)
* **添加监控项**：绑定客户归属，输入目标 `Item ID`。
* **防御规则配置**：
  * **价格防卫线**：设定竞品链接/关键词，以及触发报警的绝对低价或差价阈值。
  * **舆情防卫线**：差评检测开关及敏感词黑名单库。
  * **流量防卫线**：设定搜索关键词与目标 Zip Code，监控排名是否跌出首页。
* **调度策略分配**：支持下拉选择任务频率（T1-5分钟激进模式 / T2-30分钟常规模式）。

### 4. 🚨 战果审计与日志 (Audit 模块)
* **报警历史记录**：按客户、按 Item 筛选所有触发过的报警记录（便于截图发给客户邀功：“老板，昨晚3点帮你挡了一波恶意跟卖”）。

## 三、 技术栈选型 (现代化、易部署、易扩展)

为了达到“好看到能装逼”且底层稳定的要求，我们采用以下技术栈：

### 1. 🎨 前端展示层 (Frontend)
* **核心框架**：`Vue 3` + `Vite` (本地开发极快，打包体积小)。
* **UI 组件库**：强烈推荐 **`Tailwind CSS` + `shadcn-vue`** (或者 `Radix Vue`)。彻底抛弃 Element-UI 那种陈旧感，直接做出类似 Vercel、Stripe 那种带玻璃毛玻璃、极简暗色模式（Dark Mode）的高级质感。
* **图表库**：`ECharts` (定制暗黑主题折线图)。

### 2. 🧠 后端控制层 (Backend API)
* **核心框架**：`FastAPI` (Python)。自带互动式 API 文档，纯异步，性能极强，代码结构比 Flask 清晰百倍。
* **数据库**：**`SQLite3` (开启 WAL 模式)** + `SQLAlchemy` (ORM)。既然是单台机器自己用，完全不需要去装笨重的 MySQL/PostgreSQL，SQLite 单文件足够支撑你赚到第一个 100 万，且备份极其简单（直接拷贝数据库文件）。

### 3. 🕷️ 抓取与调度引擎 (Engine)
* **请求核心**：`curl-cffi` 模拟完美浏览器指纹 + 动态代理池接入。
* **任务调度**：`APScheduler` (集成在 FastAPI 内部，或者单独跑一个 Python 进程)。

## 四、 核心系统处理方案

### 1. 任务调度与防并发冲突
* **方案**：FastAPI 作为 Web 服务只负责响应前端页面的增删改查。后台单独起一个 `worker.py` 进程运行 `APScheduler`。两者通过 SQLite 数据库通信。
* **心跳读取**：调度器每分钟读取一次数据库里的活跃任务列表（`is_active=1` 且未到期的任务），动态加入或移除内存中的定时任务。

### 2. “拟人化”抓取与反风控
* eBay 存在极其严格的区域化展示。代码层面，抓取前必须构造特定的 Header 和 Cookie。
* `curl-cffi` 每次请求时：
  1. 随机切换 User-Agent (只用最新版的 Chrome/Edge Windows指纹)。
  2. 携带无痕会话（不保留上一轮的 Cookie）。
  3. 通过请求 eBay 的特定接口或设置特定的 `dp1` Cookie，**强行注入客户指定的美国 Zip Code**。

### 3. 报警冷却中心 (Cooldown Center) - 极其重要
* **机制**：在 SQLite 中建立 `alert_logs` 表。每次准备发企业微信/打电话前，查询最近 4 小时内，该客户的该 Item 是否触发过同类报警。
* 如果触发过 -> 仅在本地控制台打一条浅灰色的 Log（静默）。
* 如果未触发过 -> 调用企微 Webhook -> 写入 `alert_logs` -> 在控制台前端弹出醒目的红色 Toast 提示。
